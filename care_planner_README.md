# Care Planner API

Generates a **7-day personalised home activity plan** for a child, based on their age, assessment mode, and three developmental domain Z-scores. Day 1 activities are provided by the caller (from the existing activity matrix); Days 2–7 are generated by an LLM.

---

## Files

| File | Purpose |
|---|---|
| `care_planner_api.py` | FastAPI application — endpoints, request/response models, prompt builder |
| `care_planner_llm.py` | LLM abstraction — switch between OpenAI and Gemini via one env var |
| `care_planner_input.json` | Sample request body for testing |
| `care_planner_output.json` | Expected response structure (reference / mock) |
| `care_planner_README.md` | This file |

## Configuration

Create a `.env` file in the `code/` directory:

```env
# ── LLM Provider ────────────────────────────────────────────
# Set to "openai" (default) or "gemini"
LLM_PROVIDER=openai

# ── OpenAI (used when LLM_PROVIDER=openai) ──────────────────
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini          # cheapest capable model; swap to gpt-4o for higher quality

# ── Gemini (used when LLM_PROVIDER=gemini) ──────────────────
# GEMINI_API_KEY=...
# GEMINI_MODEL=gemini-1.5-flash   # cheapest Gemini model

# ── Image search — optional ──────────────────────────────────
# Free tier: 50 requests/hour. Without this key, image_url is null
# but image_keywords are always returned so you can fetch images yourself.
# UNSPLASH_ACCESS_KEY=...
```

| Provider | Model | Approx. cost per 7-day plan |

### `POST /care-plan`

Generate a 7-day activity plan.

#### Request body

```json
{
  "mode": "MONITOR",
  "age_months": 2,
  "domain1": { "name": "Motor Development", "zscore": 63.5 },
  "domain2": { "name": "Communication",     "zscore": 72.0 },
  "domain3": { "name": "Social Engagement", "zscore": 70.5 },
  "day1_activities": [
    {
      "title": "Tummy Time Reach",
      "description": "Place baby on tummy and hold a bright toy just out of reach.",
      "image_url": null
    },
    {
      "title": "Supported Sitting Wobble",
      "description": "Sit behind baby and gently tilt them side to side.",
      "image_url": null
    },
    {
      "title": "Kick & Bat",
      "description": "Hold a rattle above feet then hands and encourage kicking or batting.",
      "image_url": null
    }
  ]
}
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `mode` | string | Yes | `LEARN`, `MONITOR`, or `CONCERN` |
| `age_months` | int | Yes | 0–216 |
| `domain1/2/3` | object | Yes | `name` (string) + `zscore` (0.0–100.0) |
| `day1_activities` | array | Yes | Exactly 3 items: `title`, `description`, optional `image_url` |

#### Response body

```json
{
  "child_summary": "2-month-old in MONITOR mode ...",
  "focus_domains": ["Motor Development"],
  "plan": [
    {
      "day": 1,
      "theme": "Introduction Day",
      "activities": [
        {
          "title": "Tummy Time Reach",
          "description": "Place baby on tummy ...",
          "duration": "~1 minute",
          "steps": ["Place baby on tummy ..."],
          "tip": "Observe how your child responds ...",
          "image_url": null,
          "image_keywords": ["tummy", "time", "reach"],
          "domain_focus": "Motor Development",
          "is_priority": true
        }
      ]
    },
    {
      "day": 2,
      "theme": "Stretch & Discover",
      "activities": ["... 3 LLM-generated activities ..."]
    }
  ],
  "general_tips": [
    "Do tummy time on a firm surface only ...",
    "..."
  ]
}
```

| Field | Description |
|---|---|
| `child_summary` | Short LLM-written summary of the child's profile and plan focus |
| `focus_domains` | Domain names with Z-score < 65 (priority focus areas) |
| `plan` | Array of 7 `DayPlan` objects (Day 1 from input, Days 2–7 from LLM) |
| `plan[].theme` | Short motivating theme for the day |
| `plan[].activities[].steps` | Numbered concrete parent instructions |
| `plan[].activities[].tip` | Plain-English developmental reason for the activity |
| `plan[].activities[].image_url` | Unsplash image URL if `UNSPLASH_ACCESS_KEY` is set, else `null` |
| `plan[].activities[].image_keywords` | Always returned — use to fetch your own image |
| `plan[].activities[].is_priority` | `true` when the activity targets a domain with Z-score < 65 |
| `general_tips` | 3–5 LLM-generated tips for the whole week |

---

## Logic — how priority works

| Condition | Behaviour |
|---|---|
| Domain Z-score **< 65** | Marked as priority — LLM dedicates ≥ 2 of 3 daily activities to it |
| Domain Z-score **≥ 65** | On-track — receives 1 activity per day or is used for enrichment if all on-track |
| All domains on-track | LLM spreads activities evenly across all 3 domains |

## Switching LLM provider

Change `LLM_PROVIDER` in your `.env` file — no code changes needed.

```env
# Use OpenAI (default)
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-...

# Switch to Gemini
LLM_PROVIDER=gemini
GEMINI_API_KEY=...
```

The `/health` endpoint always shows the active provider and model.

## Testing

### Using the sample files

```bash
# Start the server
uvicorn care_planner_api:app --reload --port 8001

# Send the sample input
curl -X POST http://localhost:8001/care-plan \
  -H "Content-Type: application/json" \
  -d @care_planner_input.json
```

### Using the Python test script

```bash
python test_care_planner.py
```

The script runs three scenarios:
1. **Infant 2 months** — Motor Development is the priority domain
2. **Toddler 18 months** — Two priority domains (Motor + Communication)
3. **6 months all on-track** — Enrichment plan, all domains healthy

### Validating the output shape

Compare any live response against `care_planner_output.json` to verify all required fields are present. The actual activity content will differ between runs — only the structure should match exactly.
