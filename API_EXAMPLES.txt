5. Generate Care Plan
---------------------

Endpoint:
POST /care-plan

Description:
Generates a personalized 7-day child care activity plan. Day 1 activities are
provided by the caller. Days 2 through 7 are generated by the configured LLM.

Request:
curl -X POST "https://care-planner-api-fmp7.onrender.com/care-plan" \
  -H "Content-Type: application/json" \
  -d '{
    "mode": "MONITOR",
    "age_months": 2,
    "domain1": {
      "name": "Motor Development",
      "zscore": 63.5
    },
    "domain2": {
      "name": "Communication",
      "zscore": 72.0
    },
    "domain3": {
      "name": "Social Engagement",
      "zscore": 70.5
    },
    "day1_activities": [
      {
        "title": "Tummy Time Reach",
        "description": "Place baby on tummy and hold a bright toy just out of reach.",
        "image_url": null
      },
      {
        "title": "Supported Sitting Wobble",
        "description": "Sit behind baby and gently tilt them side to side.",
        "image_url": null
      },
      {
        "title": "Kick and Bat",
        "description": "Hold a rattle above feet then hands and encourage kicking or batting.",
        "image_url": null
      }
    ]
  }'

Required Fields
---------------

mode:
  Type: string
  Allowed values: LEARN, MONITOR, CONCERN

age_months:
  Type: integer
  Allowed range: 0 to 216

domain1, domain2, domain3:
  Type: object
  Required keys: name, zscore
  zscore range: 0.0 to 100.0

day1_activities:
  Type: array
  Requirement: exactly 3 items
  Each item requires title and description. image_url is optional.

Common Errors
-------------

Missing Gemini API key:
{
  "detail": "GEMINI_API_KEY is not set in environment / .env file."
}

Invalid request body:
FastAPI returns a 422 Unprocessable Entity response with validation details.

Common cause:
day1_activities must contain exactly 3 items.
