# care-planner-api

FastAPI service for generating a 7-day child care activity plan.

## Run locally

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp env.example .env
uvicorn care_planner_api:app --reload --port 8001
```

Open `http://localhost:8001/docs`.

## Deploy on Render

This repo includes `render.yaml`, so you can deploy it as a Render Blueprint.

1. Push this repo to GitHub.
2. In Render, choose **New +** > **Blueprint**.
3. Connect the GitHub repo and deploy the `care-planner-api` service.
4. In the service environment variables, add at least one provider key:
   - `GEMINI_API_KEY` when `LLM_PROVIDER=gemini`
   - `OPENAI_API_KEY` when `LLM_PROVIDER=openai`
5. Optional: add `UNSPLASH_ACCESS_KEY` if you want image URLs in responses.

Render health checks `/health`, and the API docs will be available at:

```text
https://<your-render-service>.onrender.com/docs
```

OpenAPI JSON is available at:

```text
https://<your-render-service>.onrender.com/openapi.json
```

Manual Render settings, if you do not use the Blueprint:

```text
Build Command: pip install -r requirements.txt
Start Command: uvicorn care_planner_api:app --host 0.0.0.0 --port $PORT
Health Check Path: /health
```

## API docs

FastAPI publishes interactive docs automatically:

- Local Swagger UI: `http://localhost:8001/docs`
- Local ReDoc: `http://localhost:8001/redoc`
- Render Swagger UI: `https://<your-render-service>.onrender.com/docs`
- Render ReDoc: `https://<your-render-service>.onrender.com/redoc`

### `GET /health`

Checks that the service is alive and shows the active LLM provider.

Example response:

```json
{
  "status": "ok",
  "service": "Neev Care Planner API",
  "llm_provider": "gemini",
  "model": "gemini-2.5-flash-lite"
}
```

### `POST /care-plan`

Generates a 7-day child care activity plan. Day 1 activities come from the request; days 2 through 7 are generated by the configured LLM.

Example request:

```bash
curl -X POST "https://<your-render-service>.onrender.com/care-plan" \
  -H "Content-Type: application/json" \
  -d '{
    "mode": "MONITOR",
    "age_months": 2,
    "domain1": { "name": "Motor Development", "zscore": 63.5 },
    "domain2": { "name": "Communication", "zscore": 72.0 },
    "domain3": { "name": "Social Engagement", "zscore": 70.5 },
    "day1_activities": [
      {
        "title": "Tummy Time Reach",
        "description": "Place baby on tummy and hold a bright toy just out of reach.",
        "image_url": null
      },
      {
        "title": "Supported Sitting Wobble",
        "description": "Sit behind baby and gently tilt them side to side.",
        "image_url": null
      },
      {
        "title": "Kick and Bat",
        "description": "Hold a rattle above feet then hands and encourage kicking or batting.",
        "image_url": null
      }
    ]
  }'
```

Required request fields:

| Field | Type | Notes |
|---|---|---|
| `mode` | string | `LEARN`, `MONITOR`, or `CONCERN` |
| `age_months` | integer | Child age from 0 to 216 months |
| `domain1`, `domain2`, `domain3` | object | Each needs `name` and `zscore` |
| `day1_activities` | array | Exactly 3 activities |

The response includes:

| Field | Notes |
|---|---|
| `child_summary` | Short summary of the child profile and focus |
| `focus_domains` | Domains with `zscore < 65` |
| `plan` | Seven daily plans, each with 3 activities |
| `general_tips` | Weekly caregiver tips |

For the full schema and live testing, use `/docs` after deployment.
